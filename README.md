How will we be editing our SVG files?​
Hey team! Now that our GitHub repository is officially up and running, we need to decide on our development workflow for our SVG assets.

Because SVGs are unique—they are both visual graphics and raw XML code—the tools we choose will change how we manage our files on GitHub.

Graphic Design Apps (Figma, Illustrator, Inkscape)​
How it works: You download/clone the SVG, open it in your favorite design software, make your visual edits, and export/overwrite the file.
Best for: Complex visual design, layout changes, and anyone who prefers a graphical interface over typing code.
Note: Design software often adds extra "metadata bloat" to the code, so we will need to use an optimizer (like SVGO or "Optimized SVG" export settings) before pushing changes to GitHub.
